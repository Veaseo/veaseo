WordPress, c'est **43% du web mondial**. Inattaquable, non ? **Faux.**

En 2025, les sites les plus performants sont massivement en train de migrer vers des stacks modernes type **Next.js**. Et ce n'est pas une mode : c'est un saut technologique qui change tout.

Voici un comparatif honnête, métier par métier, des deux approches. **Sans dénigrer aveuglément WordPress**, et sans survendre Next.js.

## WordPress : ce qu'il est vraiment

WordPress est un **CMS** (Content Management System) né en 2003. Il fonctionne en générant chaque page **à la volée** depuis une base de données MySQL au moment où un visiteur arrive.

### Ses vraies forces
- **Écosystème énorme** : 60 000+ plugins disponibles
- **Familier** des prestataires depuis 20 ans
- **Édition de contenu** accessible aux non-techniques
- **Communauté** mondiale, formations gratuites partout

### Ses vraies limites
- **Performance** : par défaut, un site WP est lent (PageSpeed 30-60% courant)
- **Sécurité** : 95% des sites piratés en 2024 étaient WordPress
- **Coût technique réel** : plugins payants, sécurité, maintenance, hébergement spécialisé
- **Dépendance** à des plugins maintenus par des tiers

## Next.js : ce que c'est, vraiment

Next.js est un **framework JavaScript** créé par Vercel en 2016, basé sur **React**. Il génère les pages **en avance** (statiquement) ou à la demande, mais avec des optimisations radicales.

### Sa philosophie
- **Static-first** : les pages sont pré-générées en HTML/CSS, livrées instantanément
- **Pas de base de données** par défaut (contenu en code ou via API)
- **Performance native** : Core Web Vitals au top, sans plugin
- **Sécurité par design** : pas de back-office exposé, surface d'attaque minimale

### Qui l'utilise ?
- **Netflix** (interface utilisateur)
- **TikTok** (web app)
- **Notion**, **Hulu**, **Stripe**
- La majorité des **startups tech récentes**
- De plus en plus d'**agences premium**

## Comparatif sur 7 critères clés

### 1. Vitesse

| | WordPress | Next.js |
|---|---|---|
| PageSpeed mobile typique | 30-70% | **90-100%** |
| LCP (Largest Contentful Paint) | 2-5s | **< 1s** |
| TTFB (Time To First Byte) | 500-2000ms | **< 100ms** |

**Verdict** : Next.js gagne haut la main. C'est le facteur n°1 pour le SEO mobile.

### 2. SEO

**WordPress** : SEO possible MAIS demande plugins (Yoast, RankMath) + configuration + optimisations techniques continues.

**Next.js** : SEO **natif** (génération statique, Schema.org facile, vitesse, structure HTML propre). Moins de configurations, plus de performance par défaut.

**Verdict** : Next.js part avec une longueur d'avance technique.

### 3. Sécurité

**WordPress** : ~95% des sites piratés du web sont WordPress. Causes :
- Plugins obsolètes ou abandonnés
- Connexions admin accessibles
- Permissions fichiers mal configurées
- Bots de spam ciblant massivement WP

**Next.js** : pas de back-office exposé, code compilé, attaques quasi-impossibles sur le site lui-même.

**Verdict** : Next.js largement plus sûr par design.

### 4. Coût total sur 3 ans

| | WordPress (estimation) | Next.js (estimation) |
|---|---|---|
| Création initiale | 2 000 - 5 000€ | 3 000 - 6 000€ |
| Hébergement (3 ans) | 200-500€/an | **0-50€/mois** (Vercel gratuit possible) |
| Plugins premium | 200-500€/an | 0€ |
| Maintenance / mises à jour | 500-1500€/an | **Quasi nulle** |
| **Total 3 ans** | ~7 500 - 12 000€ | ~3 500 - 7 000€ |

**Verdict** : Next.js gagne sur le **coût total de possession** (TCO).

### 5. Édition de contenu

**WordPress** : interface admin riche, accessible aux non-techniques. Vous publiez un article en 5 minutes.

**Next.js** : par défaut, le contenu se modifie dans le code (Markdown/MDX). Pour une vraie interface d'édition, il faut ajouter un **headless CMS** (Sanity, Strapi, Notion, etc.).

**Verdict** : WordPress gagne — pour l'instant.

### 6. Plugins et fonctionnalités

**WordPress** : 60 000+ plugins, vous trouvez tout. Mais **chaque plugin = risque sécurité + ralentissement**.

**Next.js** : pas d'écosystème plugin, mais bibliothèque NPM gigantesque. Plus de **code custom**, moins de plugins génériques.

**Verdict** : WordPress gagne en volume, Next.js en qualité.

### 7. Évolutivité technique

**WordPress** : difficile de faire « propre » au-delà d'un certain seuil de complexité. Devient vite usine à gaz.

**Next.js** : conçu pour scaler. Vous pouvez passer d'un site vitrine à une app web complète sans changer de techno.

**Verdict** : Next.js gagne sur le long terme.

## Pour qui WordPress reste pertinent

Je ne vais pas être dogmatique. WordPress est **encore le bon choix** pour :

- ✅ **Blogs très volumineux** (50+ articles/mois) avec édition non-tech
- ✅ **Sites e-commerce simples** via WooCommerce
- ✅ **Migrations de sites WP existants** quand la refonte n'est pas budgétée
- ✅ **Cas où plusieurs personnes** éditent le contenu sans formation

## Pour qui Next.js est meilleur

- ✅ **Sites vitrines** d'entreprises locales (90% des cas)
- ✅ **Refontes ambitieuses** avec exigence de performance
- ✅ **Sites où le SEO** est un enjeu majeur
- ✅ **Sites qui doivent durer 5+ ans** sans tomber dans l'obsolescence
- ✅ **Apps web** ou interfaces interactives
